Publish a Plugin

Submit your plugin to the public Zeus registry so others can discover and install it.

Quick Links

  • Registry repo: github.com/arslnb/zeus-plugins
  • Plugin directory: /zeus/plugins
  • Getting started: /zeus/docs/getting-started/
  • Manifest reference: /zeus/docs/manifest-reference/

1) Scaffold Your Plugin

git clone https://github.com/arslnb/zeus-plugins.git
cd zeus-plugins
python3 -m venv .venv
source .venv/bin/activate
python -m pip install -e .

# create scaffold
zeus plugin init acme_demo --runtime daemon --owner acme

# validate plugin
zeus plugin check plugins/acme_demo --json

2) What to Include in Your Submission

plugins/
  acme_demo/
    zeus.plugin.json
    metadata.json
    README.md
    examples/
      install.json
      config.json
    client/plugin.py   # and/or server/plugin.py

Keep your plugin id stable (folder name, manifest id, and metadata id should match).

3) Open a Pull Request

git checkout -b add-acme-demo-1-0-0
git add plugins/acme_demo
git commit -m "Add acme_demo plugin v1.0.0"
git push origin add-acme-demo-1-0-0

gh pr create --repo arslnb/zeus-plugins \
  --title "Add acme_demo plugin v1.0.0" \
  --body "Adds acme_demo plugin with install/config examples and docs."

One-Click Prompt for PR Packaging

Use this prompt with Codex or Claude to prepare a clean plugin submission.

Prepare this plugin for submission to https://github.com/arslnb/zeus-plugins.

Read these first and use them as source of truth:
- https://arslnb.com/zeus/docs/llms.txt
- https://arslnb.com/zeus/docs/agent-index.json
- https://arslnb.com/zeus/docs/plugin-registry/
- https://arslnb.com/zeus/docs/manifest-reference/
- https://arslnb.com/zeus/plugins/
- https://github.com/arslnb/zeus-plugins
- https://github.com/arslnb/zeus-plugins/tree/main/schemas

Tasks:
1) Validate zeus.plugin.json and ensure runtime entries are real.
2) Add metadata.json and matching index.json entry.
3) Add README plus examples/install.json and examples/config.json.
4) Generate git + gh commands to open a PR.

Return:
- final file tree
- full file contents
- exact PR commands
- validation command output format.

Submission Checklist

  • Manifest and metadata are valid JSON and consistent.
  • Runtime entry files exist and are executable.
  • Install/config examples are copy-paste ready.
  • README explains setup, config, and expected behavior.
  • zeus plugin check plugins/<plugin_id> --json passes.
After merge, your plugin appears in /zeus/plugins.
