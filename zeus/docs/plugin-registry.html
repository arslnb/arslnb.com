Publish a Plugin

Submit your plugin to the approved Zeus catalog so users can install it from Settings → Plugins.

Catalog Policy (Authoritative Allowlist)

  • index.json is the definitive approval list.
  • The app installs/searches only plugins listed in index.json.
  • Plugin folders not listed in index.json are ignored by catalog validation and app discovery.
  • Catalog installs track the repository main branch.

Quick Links

  • Registry repo: github.com/arslnb/zeus-plugins
  • Plugin directory: /zeus/plugins
  • Manifest reference: /zeus/docs/manifest-reference/
  • API reference: /zeus/docs/api-reference/

1) Scaffold and Validate

git clone https://github.com/arslnb/zeus-plugins.git
cd zeus-plugins
python3 -m venv .venv
source .venv/bin/activate
python -m pip install -e .

# create scaffold
zeus plugin init acme_demo --runtime daemon --owner acme

# validate plugin folder
zeus plugin check plugins/acme_demo --json

# validate allowlisted catalog
zeus plugin check . --registry --json

2) Required Submission Files

plugins/
  acme_demo/
    zeus.plugin.json
    metadata.json
    README.md
    examples/
      install.json
      config.json
    client/plugin.py   # and/or server/plugin.py
    skills/            # optional shipped skills
  • Manifest id, folder name, and metadata id must match.
  • Declare prerequisite setup in manifest (prerequisites.agent_guidance, prerequisites.cli[].install_options[]) where applicable.
  • Declare auth/setup separately in oauth; prerequisite install and auth setup are different phases now.
  • Declare shipped skills in manifest skills if you bundle them.

3) Add to Allowlist Index

Add (or update) a single entry in index.json:

{
  "id": "acme_demo",
  "name": "Acme Demo",
  "version": "1.0.0",
  "summary": "One-line summary.",
  "owner": "acme",
  "runtime": ["client"],
  "categories": ["productivity"],
  "min_zeus_version": "1.0.0",
  "source_repo": "https://github.com/arslnb/zeus-plugins",
  "license": "MIT",
  "metadata_path": "plugins/acme_demo/metadata.json",
  "manifest_path": "plugins/acme_demo/zeus.plugin.json"
}

4) Open Pull Request

git checkout -b add-acme-demo-1-0-0
git add plugins/acme_demo index.json
git commit -m "Add acme_demo plugin v1.0.0"
git push origin add-acme-demo-1-0-0

gh pr create --repo arslnb/zeus-plugins \
  --title "Add acme_demo plugin v1.0.0" \
  --body "Adds acme_demo plugin with setup/config examples and docs."

Submission Checklist

  • Manifest and metadata pass validation.
  • Index entry is present and paths are correct.
  • README explains prerequisite install flow, Zeus handoff behavior, and configuration form fields.
  • Examples include /api/plugins/install and /api/plugins/configure.
  • If secrets are needed, docs describe vault-backed configuration.
  • If prerequisites are required, provide at least one install option or docs URL for each required prerequisite.
After merge, users can install from the in-app approved catalog. If prerequisites are missing, Zeus opens a notch setup thread before handing back to plugin configuration. See the GWS plugin for the reference implementation.
