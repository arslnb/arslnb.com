Plugin API Reference

Endpoints for install/search/config, OAuth flows, server activation, channels, and custom webhooks.

Local Plugin Management (Daemon)

RouteMethodPurpose
/api/pluginsGETList local plugin state.
/api/plugins/searchPOSTSearch curated plugin catalog.
/api/plugins/installPOSTInstall from registry, git, or path.
/api/plugins/uninstallPOSTUninstall local plugin.
/api/plugins/enabledPOSTEnable/disable local plugin.
/api/plugins/configPOSTSet local plugin config and secret refs.
/api/plugins/server-statusPOSTUpdate server sync status metadata for UI/state.

Local OAuth APIs

RouteMethodPurpose
/api/plugins/oauth/startPOSTBuild consent URL and return redirect target.
/api/plugins/oauth/callbackPOSTHandle returned auth code/state and exchange tokens.
/api/plugins/oauth/refreshPOSTRefresh access token when provider supports refresh.
/api/plugins/oauth/disconnectPOSTDisconnect provider and remove token refs.

Exact availability can depend on your current daemon build/runtime flags.

Server Plugin Control Plane

All routes require signed device auth headers: X-Device-Id, X-Timestamp, X-Nonce, X-Signature.

RouteMethodPurpose
/v1/client/pluginsGETList server plugins, channel map, and webhook routes.
/v1/client/plugins/installPOSTInstall plugin from curated registry and reload runtime.
/v1/client/plugins/uninstallPOSTUninstall plugin and reload runtime.
/v1/client/plugins/enabledPOSTEnable/disable plugin and reload runtime.
/v1/client/plugins/configPOSTUpdate server plugin config/secret refs and reload runtime.
/v1/client/plugins/reloadPOSTForce controlled in-process plugin reload.

Channel and Webhook Routes

RouteMethodAuth
/v1/channels/{channel}/webhookGETNo device auth (provider verify path).
/v1/channels/{channel}/webhookPOSTNo device auth (provider ingest path).
/v1/client/channels/{channel}/sendPOSTSigned device auth required.
/webhook/{route}ANYNo device auth (plugin-defined webhook endpoint).

Common Request Payloads

Install from local path (daemon)

POST /api/plugins/install
{
  "source_type": "path",
  "source": "/absolute/path/to/plugin",
  "force": true
}

Set config + secret refs

POST /api/plugins/config
{
  "plugin_id": "acme_demo",
  "config": {
    "client": { "api_base": "https://api.acme.dev" },
    "server": {}
  },
  "secret_refs": {
    "client": { "api_key": "acme_api_key" },
    "server": { "verify_token": "acme_verify_token" }
  }
}

Install on server

POST /v1/client/plugins/install
{
  "plugin_id": "acme_channel",
  "version": "1.0.0",
  "config": {
    "server": { "endpoint_base": "https://api.acme.dev" }
  },
  "secret_refs": {
    "server": { "verify_token": "acme_verify_token" }
  }
}
For end-to-end payloads, see the example pages.
