How to Integrate with Zeus

Use these guides to connect your tools, channels, and workflows to Zeus.

Looking for Existing Plugins?

Browse community plugins at /zeus/plugins before building from scratch.

Recommended Reading Order

  1. Plugins to see what is already available.
  2. Getting Started for minimum files and first install.
  3. Build with Codex/Claude for prompt templates and workflow.
  4. Publish to Registry for the community PR workflow.
  5. Manifest Reference for full schema and policy fields.
  6. Daemon Runtime and Channel Runtime for execution behavior.
  7. API Reference for install, config, OAuth, and channel routes.
  8. Daemon Example and Channel Example.

What Zeus Plugins Can Define

Daemon Runtime

  • tools
  • commands
  • services
  • cli
  • providers
  • gateway
  • http
  • hook_phases

Server Runtime

  • Channel adapters (verify_webhook, ingest_webhook, send)
  • Custom webhooks at /webhook/{route}
  • Server-side config + secret refs
  • In-process plugin reload and activation

Behavior Extensions

  • oauth.providers for generic OAuth flows
  • automations.behaviors
  • automations.scheduled_tasks
  • proactive.screen_watch

Minimal Artifact Checklist

  • zeus.plugin.json with schema_version, id, version, components.
  • At least one entry file matching your manifest: client/plugin.py and/or server/plugin.py.
  • config_schema entries for all required runtime config/secrets.

Start with Getting Started for a copy/paste minimal scaffold.

Design Lineage

Zeus plugin architecture is loosely inspired by OpenClaw plugin concepts, but implemented as Python-first runtime contracts and Zeus-specific APIs/security semantics.

See OpenClaw Differences before porting existing plugins.

Canonical Registry Repo

The shared plugin registry lives in github.com/arslnb/zeus-plugins.

Use the same repo for schemas, scaffold/check CLI, and plugin submissions via PR.
