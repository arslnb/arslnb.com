Build Plugins with Codex and Claude Code

A practical guide to getting coding assistants to generate a Zeus plugin scaffold you can ship.

Start with One Click

These buttons copy a starter prompt, attempt to open the desktop app, and fall back to a prefilled web composer.

If deep-linking is blocked on your system, the prompt is still copied and opened in a browser tab.

Starter Prompt

Build a Zeus plugin scaffold in this repository.

Requirements:
1) Create zeus.plugin.json with schema_version, id, name, version, components, and config_schema.
2) Create matching Python entrypoint files for all declared components.
3) Implement one real end-to-end capability (no TODO placeholders).
4) Include install and config API calls I can run immediately.
5) Add a short troubleshooting section for common setup failures.

Output:
- file tree
- full file contents
- exact commands for local verification

What to Ask For

Both tools work best when you ask for concrete files and checks, not broad brainstorming.

  • Ask for a complete plugin tree with zeus.plugin.json plus entrypoints.
  • Specify daemon, channel, or hybrid plugin shape.
  • Require valid JSON and executable Python code.
  • Require test commands and expected outputs.

Prompt Template: Codex

Use this when you want Codex to build a working plugin in-place.

You are editing a Zeus plugin repo.
Create a {daemon|channel|hybrid} plugin named {plugin_id}.

Requirements:
1) Create zeus.plugin.json with schema_version, id, name, version, components, config_schema.
2) Create client/plugin.py and/or server/plugin.py to match the manifest.
3) Include one real tool or adapter flow (no TODO stubs).
4) Add a README with install/config/run instructions.
5) Validate manifest JSON and show a minimal test command.

Output:
- File tree
- Full file contents
- Commands I should run to verify the plugin locally.

Prompt Template: Claude Code

Use this when you want a more detailed implementation plan before edits.

Build a Zeus plugin from scratch in this repository.

Plugin shape: {daemon|channel|hybrid}
Plugin id: {plugin_id}
Primary capability: {what it should do}
External API dependencies: {none or list}

Deliverables:
- zeus.plugin.json
- entry files for all declared components
- config_schema + secret_refs guidance
- one end-to-end happy path
- troubleshooting notes for common setup failures

Constraints:
- No placeholder pseudocode
- Keep handlers small and testable
- Return exact commands for install and config API calls.

Install/Config API Calls You Can Reuse

Install from local path

POST /api/plugins/install
{
  "source_type": "path",
  "source": "/absolute/path/to/my_plugin",
  "force": true
}

Set config and secret refs

POST /api/plugins/config
{
  "plugin_id": "acme_demo",
  "config": {
    "client": { "api_base": "https://api.acme.dev" },
    "server": {}
  },
  "secret_refs": {
    "client": { "api_key": "acme_api_key" },
    "server": {}
  }
}

Server install for server component

POST /v1/client/plugins/install
{
  "plugin_id": "acme_demo",
  "version": "1.0.0",
  "config": { "server": {} },
  "secret_refs": { "server": {} }
}

Use the Shared CLI + Schemas

Ask agents to work inside arslnb/zeus-plugins so scaffold and validation are standardized.

zeus plugin init my_plugin --runtime daemon --owner yourname
zeus plugin check plugins/my_plugin --json

Quality Checklist Before You Ship

  • Manifest fields match actual entry files.
  • All required config values and secret refs are documented.
  • Tool/adaptor output shape is deterministic and parseable.
  • Failure paths return useful errors, not raw tracebacks.
  • Install + config requests succeed on a clean environment.

Recommended Workflow

  1. Generate first scaffold with Codex or Claude Code.
  2. Run install/config calls and capture exact failures.
  3. Feed failures back into the model with logs and file paths.
  4. Repeat until one happy path works end-to-end.
  5. Only then add optional hooks, OAuth, and runtime policy controls.
Tip: keep your first plugin narrow. One reliable workflow beats ten partial ones.
